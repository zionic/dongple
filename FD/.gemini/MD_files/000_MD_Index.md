# 동플허브_문서_인덱스

## 기준 문서
- `동플허브_마스터스펙_구현.md`
- 역할: 동플 허브의 마스터 스펙이자 단일 기준 문서
- 포함 범위: 제품 목표, 아키텍처, 데이터 모델, 인증/익명화, 분류/신뢰도 정책

## 문서 사용 원칙
1. 모든 기획, 구현, 분석은 먼저 `동플허브_마스터스펙_구현.md`를 확인한다.
2. 핵심 정책이 바뀌면 먼저 기준 문서를 수정하고, 이후 상세 문서를 동기화한다.
3. 나머지 문서는 기준 문서를 보완하는 상세 설계서, 작업 기록서, 전환 계획서로 분류한다.

## 추천 읽기 순서
1. `동플허브_마스터스펙_구현.md`
2. 작업하려는 주제에 맞는 상세 문서 1개 이상
3. 구현 시 관련 SQL(`schema.sql`, `seed_suwon_events.sql`) 확인

## 문서 운영 규칙
- 기준 문서와 상충하는 내용이 있으면 기준 문서를 우선한다.
- 상세 문서는 "왜/어떻게 구현하는지"를 구체화하고, 작업 기록 문서는 "무엇을 시도했는지"를 남긴다.
- 추후 새 문서를 추가할 때는 이 인덱스에 먼저 등록한다.

## 현재 정리 상태 메모
- 현재 폴더는 앱 소스 저장소가 아니라 문서/DB 초안 중심의 작업 폴더이다.
- 일부 문서가 참조하는 `src/...` 경로는 현재 폴더에 없으며, 별도 앱 저장소 또는 후속 구현 대상으로 본다.
- `schema.sql`은 마스터 스펙 전체가 아니라 현재까지 정리된 MVP/초기 구현 범위를 반영한다.
- 지침 문서는 한글 작성 원칙을 적용한다.
