# AGENTS

## 기본 원칙
- 현재 프로젝트 범위의 `AGENTS.md`를 우선 적용한다.
- 모든 작업과 분석은 먼저 `동플허브_마스터스펙_구현.md`를 확인한 뒤 시작한다.
- 기준 문서와 다른 문서가 충돌하면 `동플허브_마스터스펙_구현.md`를 우선한다.

## 기준 문서

## 문서 해석 규칙
- `계획`: 앞으로 할 일, 전환 순서, 실행 예정 작업
- `구현`: 이미 정리된 설계, 현재 구조, 구현 기준, 결과물 설명
- `보완`: 기존 기능 개선, 고도화, 연동 확장, 품질 향상

## 문서 작성 규칙
- 지침서, 설계서, 계획서, 보완 문서는 기본적으로 한글로 작성한다.
- 제목, 섹션명, 설명 문장은 한글 우선을 원칙으로 한다.
- 코드, API 경로, SQL, 라이브러리명, 외부 서비스명 같은 고유 기술 표기는 원문을 유지할 수 있다.
- 기존 문서에 영어 표현이 남아 있더라도, 이후 수정 시에는 한글 표현으로 점진적으로 통일한다.
## 작업 절차
- 작업 전 `동플허브_마스터스펙_구현.md`에서 관련 정책, 아키텍처, 데이터 모델을 먼저 확인한다.
- 작업 주제에 따라 아래 순서로 연관 문서를 확인한다.
- 전환/로드맵 관련 작업: `동플허브_V2메인전환_계획.md`
- 1차 구현 착수 기준 확인: `동플허브_1차구현지시서_계획.md`
- 구현 에이전트 전달 프롬프트 확인: `동플허브_에이전트프롬프트_계획.md`
- 시스템 구조 설계 확인: `동플허브_시스템설계_구현.md`
- 백엔드/DB/API/권한 설계 확인: `동플허브_백엔드DBAPI권한_구현.md`
- 프론트 구조 설계 확인: `동플허브_프론트구조_구현.md`
- TourAPI 상세 설계 확인: `동플허브_tourapi_구현.md`
- TourAPI 확장 및 연동 강화: `동플허브_tourapi_보완.md`
- 메인페이지 개선: `동플허브_메인페이지_보완.md`
- 실시간 상태 보드 개선: `동플허브_실시간상태보드_보완.md`
- 지도 화면 개선: `동플허브_지도화면_보완.md`

## 문서 관리 규칙
- 새 문서를 추가할 때는 먼저 마스터 스펙과의 관계를 확인한다.
- 정책이나 공통 규칙이 바뀌면 `동플허브_마스터스펙_구현.md`를 먼저 수정한다.
- 세부 기능 변경은 해당 `계획`, `구현`, `보완` 문서에 반영한다.
- 파일명 규칙은 `도메인_주제_문서성격.md`를 사용한다.
