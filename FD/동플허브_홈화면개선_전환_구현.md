# 동플 UX 리팩토링 지시서 (홈 → 지도 전환)

## 1. 목적

홈 화면과 지도 화면을 분리된 기능이 아니라
**연결된 하나의 흐름으로 동작하도록 개선한다.**

핵심 목표:

* 홈 = 요약
* 지도 = 확장
* 유저가 자연스럽게 이동
* 클릭/탭 시 맥락 유지

---

## 2. 핵심 UX 원칙

1. 홈은 상태를 보여주는 화면이다 (읽기)
2. 지도는 탐색과 행동 화면이다 (탐색 + 공유)
3. 모든 전환은 “맥락 유지”가 되어야 한다
4. 유저는 “이동했다”가 아니라 “확장됐다”라고 느껴야 한다

---

## 3. 기능 요구사항

---

### 3.1 카드 → 지도 전환 (필수)

#### 동작

홈에서 장소 카드 클릭 시:

* 지도 화면으로 이동
* 해당 장소로 카메라 이동
* 해당 마커 강조 표시
* 바텀시트 자동 오픈

---

#### UX 흐름

홈 카드 클릭
→ 지도 화면 전환
→ 지도 카메라 이동 (애니메이션)
→ 마커 강조
→ 바텀시트 표시

---

#### 구현 요구사항

* route 이동 시 place_id 전달
* 지도 초기화 후 camera move 실행
* animation duration: 300~500ms
* zoom level: 해당 장소 식별 가능 수준

---

#### 바텀시트 상태

자동 오픈:

```
성수 팝업

🔴 지금 붐빔 (3분 전)
대기 김 / 사람 많음

[지금 공유]
```

---

### 3.2 “지도에서 전체 보기” 버튼

#### 위치

홈 하단 CTA

---

#### 동작

버튼 클릭 시:

* 지도 화면으로 이동
* 현재 위치 기준으로 카메라 설정
* 주변 마커 전체 노출

---

#### 구현 요구사항

* geolocation 기반 중심 좌표 설정
* default zoom level 설정
* 바텀시트는 닫힌 상태 유지

---

### 3.3 공유 버튼 흐름 유지

#### 규칙

홈에서는 공유를 직접 수행하지 않는다.

---

#### 예외 (허용)

* 카드 내 공유 버튼 → 해당 장소 기준 공유
* 하단 FAB → 현재 위치 기준 공유

---

#### 필수 조건

공유 시 항상:

* 장소 정보 먼저 표시
* 그 다음 상태 선택

---

### 3.4 지도 진입 시 애니메이션 (중요)

#### 요구사항

* 화면 전환 시 자연스러운 애니메이션
* abrupt change 금지

---

#### 애니메이션 규칙

* fade + slide 조합
* map camera move 포함
* marker highlight 효과

---

### 3.5 상태 강조 처리

지도 진입 시:

* 선택된 장소 마커 확대
* 색상 강조 유지
* 다른 마커는 기본 상태 유지

---

## 4. 비기능 요구사항

* 전환 시간 500ms 이하 유지
* 초기 로딩 시 skeleton UI 제공
* 지도 렌더링 지연 최소화

---

## 5. 금지 사항

다음은 절대 구현하지 않는다:

* 홈에서 직접 상태 입력 UI 제공
* 지도 이동 없이 화면만 전환
* 바텀시트 없이 상세 페이지 이동
* 이벤트/장소 상세 텍스트 과다 노출

---

## 6. 기대 결과

* 홈 → 지도 전환 시 UX 끊김 제거
* 사용자 위치/맥락 유지
* 공유 행동 자연스럽게 증가
* 지도 사용 비율 상승

---

## 7. 핵심 요약

홈은 “지금 상황을 보여주는 요약 화면”이고
지도는 “그 상황을 확인하고 행동하는 확장 화면”이다.

두 화면은 분리된 기능이 아니라
하나의 흐름으로 동작해야 한다.
