# 동플허브_문서운영_스킬

## 목적
이 스킬은 `동플허브_마스터스펙_구현.md`를 기준으로 동플 허브 관련 문서와 구현 방향을 일관되게 해석하고 정리하기 위한 로컬 작업 규칙이다.

## 기준 문서
- 우선 확인: `C:\dongple_workspace\dongple\FD\동플허브_마스터스펙_구현.md`
- 보조 인덱스: `C:\dongple_workspace\dongple\FD\.gemini\MD_files\000_MD_Index.md`

## 문서 역할 정의

### 마스터 스펙
- `동플허브_마스터스펙_구현.md`
- 제품/도메인 레벨 규칙, 아키텍처, 데이터 모델, 인증/익명화, 분류 정책의 기준

### 확장 스펙
- `동플허브_tourapi_보완.md`
- `동플허브_tourapi_구현.md`
- 외부 데이터 연동, 신뢰도 보정, 확장 DB 구조를 설명

### 실행 문서
- `동플허브_메인페이지_보완.md`
- `동플허브_실시간상태보드_보완.md`
- `동플허브_지도화면_보완.md`
- 실제 화면/UX 개선 아이디어와 작업 히스토리

### 전환 계획
- `동플허브_V2메인전환_계획.md`
- 라우팅, 페이지 교체, 릴리스 순서를 정의

## 작업 절차
1. 먼저 `동플허브_마스터스펙_구현.md`에서 현재 작업이 어느 모듈에 속하는지 확인한다.
2. 해당 모듈과 연결된 상세 문서를 인덱스에서 찾는다.
3. 구현/수정 시에는 기준 문서와 상세 문서가 서로 충돌하지 않는지 검토한다.
4. 도메인 정책이 바뀌면 기준 문서를 먼저 갱신한다.
5. 화면 세부 구현이나 진행 이력은 작업 기록 문서에 남긴다.

## 문서 업데이트 규칙
- 아키텍처, 데이터 모델, 정책 변경: `동플허브_마스터스펙_구현.md` 우선 수정
- 특정 기능 연동 변경: 해당 상세 스펙 문서 수정
- UI 구현 변화: 작업 기록 문서 또는 별도 UI 스펙 문서 수정
- 새 문서 추가 시: `000_MD_Index.md`에 반드시 등록

## 추천 분류 기준
- "무엇을 만드는가": 마스터 스펙
- "어떻게 붙이는가": 연동/상세 스펙
- "화면에서 어떻게 보이는가": UI 작업 기록
- "언제 어떻게 전환하는가": 전환 계획

## 현재 워크스페이스 메모
- 현재 폴더는 앱 소스 저장소가 아니라 문서/DB 초안 중심의 작업 폴더이다.
- 일부 문서가 참조하는 `src/...` 경로는 현재 폴더에 없으며, 구현 대상 메모 또는 별도 저장소 기준 경로로 해석한다.
- `schema.sql`은 마스터 스펙 전체가 아니라 현재까지 정리된 초기 구현 범위를 반영한다.
- 지침 문서는 한글 작성 원칙을 따른다.
