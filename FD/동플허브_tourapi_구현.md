# [동플 허브] TourAPI 4.0 통합 상세 설계서 (FD)

본 문서는 한국관광공사의 TourAPI 4.0(KorService1)을 동플 허브 시스템에 통합하기 위한 상세 기술 설계 및 사용자 경험(UX) 가이드를 제공합니다.

---

## 1. 시스템 아키텍처 (System Architecture)

![TourAPI 시스템 플로우](C:\Users\tstar\.gemini\antigravity\brain\813093a6-f226-489d-aedd-dd99b2f2106a\tourapi_system_flow_1776289289224.png)

TourAPI 통합은 크게 **수집(Collector)**, **분류(Classifier)**, **메타데이터 강화(Meta Enrichment)**의 3단계로 구성됩니다.

### 1.1 데이터 수집 (Collector)
- **대상 API**: `searchFestival1` (축제/행사)
- **주기**: 매일 00:05 (KST)
- **로직**:
  - `eventStartDate`를 당일로 설정하여 호출.
  - 응답 데이터의 `raw_data` 전체를 `raw_items` 테이블에 JSONB 형태로 영구 보존.
  - API 호출 실패 시 **Exponential Backoff** 기반 3회 재시도. 최종 실패 시 Slack 알림 발송.

### 1.2 데이터 분류 및 매칭 (Classifier)
- **로직**:
  - 수집된 `FestivalItem`의 좌표(`mapx`, `mapy`)를 기준으로 기존 `events` 테이블과 Geofencing 매칭(반경 50m).
  - 매칭 성공 시:
    - 해당 이벤트의 `trust_score`를 **+0.2 가산** (최대 1.0).
    - 공식 명칭(`title`) 및 좌표로 마스터 데이터 보정.
    - `events_ext` 테이블에 `ext_id(contentid)`를 연결하고 전체 메타데이터 업데이트.
  - 매칭 실패 시:
    - 신규 **공식 이벤트**로 `events` 테이블에 자동 등록(선택 사항).

### 1.3 메타데이터 강화 (MetaService)
- **대상 API**: `detailWithTour1` (무장애), `detailPetTour1` (반려동물)
- **기능**:
  - `contentid`를 기반으로 상세 편의시설 정보 조회.
  - `events_ext`의 `meta` 필드에 해당 정보를 구조화하여 저장. UI 노출 플래그 설정.

## 2. 사용자 경험 및 인터페이스 (UX/UI Design)

![TourAPI UI 모의 시안](C:\Users\tstar\.gemini\antigravity\brain\813093a6-f226-489d-aedd-dd99b2f2106a\tourapi_ui_mockup_1776289271976.png)

### 2.1 신뢰도 뱃지 (Trust Badge)
- TourAPI 공식 데이터와 매칭된 제보 콘텐츠에는 **"공식 인증 정보"** 뱃지를 노출하여 사용자 신뢰도를 극대화합니다.
- 녹색 체크마크와 함께 "한국관광공사 제공" 출처를 명시합니다.

### 2.2 특화 정보 아이콘 (Special Feature Icons)
- **무장애(Barrier-free)**: 휠체어 접근 가능, 장애인 주차장, 엘리베이터 유무 등을 아이콘으로 직관적 표시.
- **반려동물(Pet-friendly)**: 반려동물 동반 가능 여부 및 관련 시설(배변봉투, 쉼터 등) 아이콘 노출.

### 2.3 시각적 즉시성 (Visual Hook)
- **실시간 혼잡도 시각화**: 지도 위 컬러풀한 히트맵이나 펄스(Pulse) 애니메이션을 통해 지역별 활성도를 즉각적으로 전달합니다.
- **감성적 데이터 메시징**: "현재 3,500명"과 같은 수치 제공에서 나아가, "지금 경복궁은 산책하기 딱 좋은 여유로운 상태예요 🌿"와 같이 관광 데이터와 사진을 결합한 매력적인 상태 정보를 제공합니다.

### 2.4 실시간 희소성 및 가이드 (Urgency & Guide)
- **희소성 강조**: "지금 이 순간에만 볼 수 있는 핫플레이스" 정보를 실시간 트렌드와 결합하여 노출, 사용자의 지속적인 접속을 유도합니다.
- **초보자 가이드**: 데이터가 낯선 신규 사용자를 위해 "현재 가장 인기 있는 여행 여정"이나 "실시간 집계 핫스팟" 추천을 제공하여 서비스 진입 장벽을 낮춥니다.

## 3. 데이터 베이스 설계 (Database Schema)

```sql
-- 이벤트 확장 정보 (TourAPI 연동 핵심)
CREATE TABLE public.events_ext (
    event_id BIGINT REFERENCES public.events(id) ON DELETE CASCADE PRIMARY KEY,
    source VARCHAR(50) DEFAULT 'TOURAPI',
    ext_id VARCHAR(100), -- TourAPI contentId
    meta JSONB DEFAULT '{}'::jsonb, -- 무장애, 반려동물 등 상세 정보
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 원본 데이터 보존용
CREATE TABLE public.raw_items (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    source VARCHAR(50) NOT NULL,
    raw_data JSONB NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 4. 향후 확장성 (Future Scalability)

- **실시간 트렌드 분석**: `getAreaBasedList1`을 연동하여 특정 지역의 관광 수요와 실시간 제보 밀도를 결합한 "혼잡도 테라피" 서비스 제공.
- **이미지 최적화**: TourAPI의 `firstimage`를 CDN(S3/CloudFront)을 통해 리사이징 및 캐싱하여 모바일 트래픽 절감.

## 5. 여정의 앨범 (Journey's Album)
사용자가 앱 내에서 탐색한 흔적을 '디지털 전리품'으로 만들어 제공함으로써 소유 효과(Endowment Effect)를 자극하고 공유를 통한 유입을 유도합니다.

### 5.0 메뉴 배치 원칙
- 앨범은 `나의 동플`에 포함한다.
- 앨범 진입은 `나의 동플 > 내 앨범` 경로로 제공한다.
- 이유: 전면 UX의 중심을 `상황판/제보/지역`에 두고, 앨범은 개인화 기능으로 맥락화하기 위함.

### 5.1 앨범 화면 구성 레이아웃

| 섹션 명칭 | 내용 및 시각 요소 | 사용자 기대 반응 |
| :--- | :--- | :--- |
| **Intro: 오늘의 발자국** | 탐색한 경로를 지도 위에 실선으로 부드럽게 드로잉하는 애니메이션 | "내가 오늘 이렇게 움직였네?"(성취감) |
| **Best Moment: 최고의 순간** | 관심도가 가장 높았던 관광지의 사진과 데이터 카드 | "여기가 제일 핫했구나!"(정보 확인) |
| **Timeline: 시간의 흐름** | 타임 슬라이더로 데이터 변화를 보여주는 짧은 루프 애니메이션 | "시간대별 차이가 크네?"(통찰력) |
| **Ending: 한 줄 브리핑** | 사용자 탐색 패턴 분석 ("활기찬 장소를 좋아하는 탐험가예요!") | "나를 알아봐 주네?"(친근함/개인화) |

### 5.2 활용 전략
- **이미지 저장 및 공유**: 생성된 앨범을 이미지로 저장하거나 SNS에 공유할 수 있는 기능을 제공하여 브랜딩 노출을 극대화합니다.
- **수집 욕구 자극**: 방문 지역별 앨범 컬렉션을 제공하여 사용자의 재방문을 유도합니다.
