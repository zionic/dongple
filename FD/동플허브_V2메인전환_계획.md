# V2 메인 페이지 정식 전환 계획

이 문서는 테스트 중인 프리미엄 V2 페이지를 서비스의 기본 루트(/) 페이지로 완전히 전환하기 위한 단계별 계획을 담고 있습니다.

## 1. 현재 상태 (테스트 중)
- **임시 조치**: `src/app/page.tsx` 상단에 `redirect("/v2")`가 추가되어 있습니다.
- **결과**: 사용자가 `/` 접속 시 자동으로 `/v2`로 이동합니다. 기존의 레거시 코드는 `page.tsx` 내에 그대로 보존되어 있습니다.

## 2. 정식 전환 절차 (전원 승인 시)

### 단계 1: 파일 교체 (Root Page)
- `src/app/page.tsx`의 모든 내용을 삭제합니다.
- `src/app/v2/page.tsx`의 내용을 `src/app/page.tsx`로 복사합니다.
- 컴포넌트 이름을 `HomeV2`에서 `Home`으로 수정합니다.

### 단계 2: 중복 경로 정리
- `src/app/v2/` 디렉토리 전체를 삭제합니다. (불필요한 중복 코드 제거)

### 단계 3: 레이아웃 최적화
- `V2` 페이지 내의 `<style jsx global>` 중 기존 헤더/푸터를 숨기는 로직을 검토합니다.
- 만약 모든 페이지가 V2 스타일로 통합된다면, `src/app/layout.tsx`에서 구형 헤더를 제거하고 V2 헤더를 글로벌로 승격시킬 수 있습니다.

## 3. 리스크 및 복구 대책
- **복구**: 전환 전 `src/app/page.tsx`를 `src/app/page.legacy.tsx`로 이름을 변경하여 백업해 둠으로써 문제 발생 시 즉각 복구가 가능하도록 합니다.
- **검색 엔진(SEO)**: `/v2` 주소가 검색 엔진에 이미 색인된 경우, 정식 전환 후 `/v2`에서 `/`로 301 리다이렉트를 유지하는 것이 권장됩니다.
