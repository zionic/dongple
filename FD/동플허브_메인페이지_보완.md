## 1️⃣ 현재 페이지가 겪는 대표적인 문제점

| 영역 | 흔히 나타나는 증상 | 왜 문제가 되는가 |
|------|-------------------|------------------|
| **​핵심 CTA 부재​** | “시작하기 / 가입하기” 버튼이나 안내가 흐릿하거나 페이지 하단에 숨겨짐 | 사용자는 **​뭘 해야 할지​** 바로 파악하지 못해 이탈률이 높아짐 |
| **​이미지·리소스 로딩 지연​** | 대형 배너·아이콘이 전부 순차적으로 로드 → LCP > 2.5 s | 페이지가 **​무겁게​** 느껴져 전환·SEO에 부정적 영향 |
| **​코드 번들 과다​** | 모든 컴포넌트가 한 번에 로드 → TBT > 300 ms | 브라우저가 메인 스레드를 오래 차단해 **​상호작용 지연​** 발생 |
| **​폰트·스타일 최적화 부족​** | 기본 weight·렌더링 차단, 한글 폰트가 흐릿함 | **​FOUT/FOIT​** 현상이 눈에 띄고 모바일 가독성이 떨어짐 |
| **​메타·SEO 데이터 부재​** | `<title>`, `<meta description>` 혹은 구조화 데이터가 없거나 빈값 | 검색 엔진 노출이 낮아 **​신규 유입​**이 제한 |
| **​접근성(ARIA·Alt) 미흡​** | 이미지에 alt 텍스트 없거나 색 대비 낮음 | WCAG AA 미준수 → 법적·윤리적 리스크와 **​사용자 만족도​** 감소 |
| **​모바일·반응형 디자인​** | 고정 폭 레이아웃, 터치 영역 작음 | 모바일 사용자가 **​불편​**하게 느끼면 즉시 탈출 |
| **​신뢰 요소(사회증명) 부재​** | 실제 사용자 사진·후기, 파트너 로고, ‘몇 천 명이 사용 중’ 같은 지표 없음 | 사용자는 **​신뢰성​**을 판단하기 어려워 바로 전환하지 않음 |
| **​분석·피드백 루프​** | 이벤트 트래킹 없이 전환률을 알 수 없음 | 개선이 **​정량화​**되지 않아 효율 저하 |

## 2️⃣ 구체적인 개선 포인트 & 적용 예시

### 2‑1️⃣ 핵심 가치·CTA를 눈에 띄게

| 개선 | 구현 예시 | 기대 효과 |
|------|----------|-----------|
| **​헤더에 ‘동플 바로!’ 슬로건을 크게​** | ```jsx\n<header className=\"hero\">\n  <h1 className=\"hero__title\">동플 바로! 우리 동네 순간을 기록하세요</h1>\n  <button className=\"hero__cta\" onClick={handleSignup}>시작하기</button>\n</header>\n``` | 사용자가 페이지 진입 직후 **​핵심 가치​**를 파악하고 행동 유도 |
| **​CTA 버튼을 고정·눈에 띄게​** | CSS `position: sticky; bottom: 0;` 혹은 `margin-top: -2rem;` | 스크롤을 하지 않아도 언제든 **​전환 기회​** 확보 |
| **​사회적 증명(뱃지·숫자) 삽입​** | ```jsx\n<p className=\"hero__badge\">30,000+ 사용자가 이미 기록 중</p>\n``` | 초기 진입 장벽을 낮추고 **​신뢰도​** 상승 |

### 2‑2️⃣ 이미지·리소스 최적화 (DevOcean 가이드라인 적용)

| 개선 | 적용 방법 | KPI 변화 (예시) |
|------|----------|----------------|
| **​이미지 포맷 교체​** (PNG/JPEG → WebP/AVIF) | 빌드 단계에서 `sharp`/`imagemin` 플러그인 사용 <br>```js\nimport imagemin from 'imagemin'; await imagemin([srcPath], {destination: destPath, plugins: [imagemin.webp()]});\n``` | 파일 용량 30\~80 % 감소 → **​LCP 15 % 개선​** |
| **​Lazy‑load 도입​** | `<img src=\"placeholder.png\" loading=\"lazy\" alt=\"…\">` 혹은 IntersectionObserver | 비핵심 이미지 로드 시점이 뒤로 미뤄져 **​FCP·LCP​** 모두 빨라짐 |
| **​이미지 크기 사전 지정​** | CSS `width: 100%; height: auto;` + `srcSet` 활용 | 불필요한 레티나 이미지 로드 방지 → **​네트워크 트래픽 감소​** |
| **​이미지 CDN 활용​** | Vercel Edge Functions + `next-optimized-images` 혹은 Cloudflare 이미지 변환 | 전 세계 사용자에게 **​빠른 전달​** (TTFB ↓) |
| **​불필요한 배경 이미지 제거​** | 실제 UI에 사용되지 않는 `src=\"\"` 삭제 | **​불필요한 요청​** 0 → **​FCP​** 개선 |

### 2‑3️⃣ 코드 번들·스크립트 최적화

| 개선 | 적용 방법 | 기대 효과 |
|------|----------|-----------|
| **​코드 스플리팅​** (React.lazy + Suspense) | ```jsx\nconst MomentList = React.lazy(() => import('./components/MomentList'));\n``` | 초기 번들에 **​MomentList​** 미포함 → **​TBT ↓​** |
| **​`defer`/`async` 속성​** 적용 | `<script src=\"/static/js/main.js\" defer></script>` | 렌더링 차단 감소 → **​FCP ↓​** |
| **​핵심 스크립트 Preload​** | `<link rel=\"preload\" href=\"/static/js/vendor.js\" as=\"script\">` | 중요한 외부 라이브러리 우선 로드 |
| **​불필요한 polyfill 제거​** | `core-js` 번들 최소화, `babel-preset-env` 사용 | 파일 크기 **​\~10 %​** 절감 |

### 2‑4️⃣ 폰트·스타일 최적화

| 개선 | 구현 | 효과 |
|------|------|------|
| **​WOFF2 폰트 사용​** + **​Weight 최소화​** | ```css\n@font-face {\n  font-family: 'Nanum Gothic';\n  src: url('/fonts/NanumGothic.woff2') format('woff2');\n  font-weight: 400, 700;\n}\n``` | 브라우저가 **​빠른 폰트 로딩​** 가능 → FOUT 최소화 |
| **​CSS `font-display: swap`​** | `@font-face {font-display: swap;}` | 텍스트가 **​즉시 표시​** (FOIT 방지) |
| **​불필요한 폰트 패밀리 제거​** | `import './fonts/NanumGothic.css';` (필요 시 로드) | **​TBT​** 감소, 모바일 CPU 부하 감소 |

## 3️⃣ 개선 로드맵 (우선순위)

| 단계 | 목표 | 주요 작업 | 기대 KPI |
|------|------|-----------|----------|
| **​1️⃣ 빠른 Wins​** (1 ~ 2 일) | 기본 접근성·CTA·이미지 lazy‑load 적용 | - `loading=\"lazy\"` 적용<br>- CTA 고정·버튼 색 대비 개선<br>- 메타·Open Graph 삽입 | FCP ↓ 30 % , CPA ↓ 10 % |
| **​2️⃣ 성능 최적화​** (1 ~ 2 주) | LCP < 2.5 s, TBT < 200 ms | - 이미지 WebP/AVIF 변환<br>- 이미지 CDN 설정<br>- 코드 스플리팅 & `defer` 적용 | LCP ↓ 15 % , TBT ↓ 45 % |
| **​3️⃣ UI/UX 리디자인​** (2 ~ 4 주) | 가치 전달 명확화, 전환률 ↑ | - Hero 레이아웃 재설계<br>- 사회적 증명 섹션 추가<br>- 모바일 터치 영역 확대 | 전환율 ↑ 20 % (예상) |
| **​4️⃣ SEO·구조화 데이터​** (3 ~ 5 주) | 검색 유입 증가 | - Title, Description, OG, JSON‑LD 삽입<br>- 사이트맵 자동 생성 | 검색 트래픽 ↑ 15 % |
| **​5️⃣ 분석·A/B​** (5 ~ 8 주) | 데이터 기반 지속적 개선 | - GA/Clarity 연동<br>- 2\~3가지 CTA·문장 A/B 테스트 | 전환율 최적화 (실험당 1\~3 % 상승) |

> **​Tip:​** 가장 큰 ROI는 **​이미지·코드 · 폰트 최적화​**와 **​핵심 CTA 강화​**에 있습니다. 이 두 가지만 해결해도 페이지 로드 속도가 눈에 띄게 빨라지고, 사용자는 “시작하기” 버튼을 바로 클릭하게 됩니다.

## 4️⃣ 구현 예시 (마크업·코드 스니펫)

### 4‑1️⃣ 이미지 최적화 파이프라인 (Next.js + imagemin)

```js
// next.config.js
module.exports = {
  images: {
    loader: 'custom',
    path: '',
  },
};

// lib/imageOptimizer.js
import sharp from 'sharp';
import path from 'path';
import fs from 'fs-extra';

export async function optimizeImage(src, dest) {
  await sharp(src)
    .resize({ fit: 'contain', width: 3000 })   // 최대 3000px
    .toFormat('webp', { quality: 80 })
    .toFile(dest);
}
