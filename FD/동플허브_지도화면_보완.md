# 동네 지도(Map) 뷰 고도화 작업 기록

## 1. 개요
* **작업 시점**: 2026-03-24
* **핵심 목적**: `LiveStatusBoard`의 "더 많은 동네 상황 보기" 기능에서 이어지는 공간 기반 리스트 뷰 구축 및 UX/UI 모바일 최적화.
* **주요 작업 파일**: `src/app/map/page.tsx`

---

## 2. 세부 기능 구현 내역

### 1) 바텀 시트(Bottom Sheet) 기반 스크롤 리스트 구현
지도 위에 둥둥 떠 있는 확장형 바텀 시트를 도입하여 공간 활용도 극대화.
* **스크롤(`overflow-y-auto`) 처리**: 장소가 아무리 늘어나도 부팅된 화면의 틀을 깨지 않고 바텀 시트 내부에서만 깔끔하게 스크롤됨.
* **자유로운 높이 조절(Drag & Snap)**: 상단 핸들바를 드래그하여 시트의 높이를 15vh(최소) ~ 50vh(절반) ~ 92vh(전체) 세 가지 상태로 부드럽게 스냅핑(Snapping)하며 조절 가능. (Pointer Event 기반 구현)

### 2) 지도 화면 내 [요청/공유] 폼 통합
유저가 지도를 탐색하다가 메인으로 돌아갈 필요 없이 즉각적으로 새로운 상태를 갱신하거나 질문을 던질 수 있도록 설계.
* 바텀 시트 상단 헤더에 `[요청(갈색)]`, `[공유(녹색)]` 버튼을 배치.
* 버튼 클릭 시 지도 위쪽으로 메인 화면과 완벽히 동일한 모달 팝업이 등장하며 폼(Form)을 제출할 수 있음.
* 제출 즉시 리스트(`localCards`)와 지도 상의 핀(`markers`) 배열 모두 상태가 자동 갱신되는 반응형 상태 관리 구현.

### 3) 팝업 대신 아코디언(Accordion) 방식의 히스토리 뷰
카드를 클릭했을 때 화면을 다시 덮는 모달(Modal) 대신, 카드 하단 영역이 부드럽게 세로로 열리는 아코디언 UI를 도입.
* 불필요한 Depth 확장을 방지하고 사용자의 인지 흐름을 유지.
* 펼쳐진 영역에는 해당 장소의 '상황 변동 이력(Timeline)'을 시계열로 표시.
* 아코디언 영역 내부에도 `[이웃에게 묻기]`, `[다른 상황 제보]` 버튼을 내장하여 언제든 즉각적인 액션 유도.

### 4) 지도 마커 인터랙션 및 양방향 자동 스크롤 연동
사용자가 리스트 카드를 찾기 위해 강제로 스크롤해야만 하는 불편함을 제거하고 지도의 공간적 이점을 활용.
* **마커 시각적 강조**: 지도 위의 핀(Marker)을 클릭하면 핀 크기가 커지고 "여기 있어요!" 말풍선 및 파동(Pulse) 애니메이션이 재생되어 명확한 시각 피드백 제공.
* **오토 스크롤(scrollIntoView)**: 마커를 클릭하면 숨겨져있던 바텀 시트가 알아서 50% 높이로 올라오고, 내부의 복잡한 스크롤 컨테이너를 통제해 해당 장소의 리스트 카드를 화면 정중앙으로 스무스하게 끌고 옴. 
* 연동되어 아코디언(히스토리)이 즉시 펼쳐지는 완벽한 양방향 동기화 완성.

## 3. 백엔드(API) 연동을 위한 향후 태스크
현재는 지도 좌표 데이터(`top`, `left`)와 리스트 정보(`localCards`)가 Mock 데이터로 시뮬레이션 중입니다.
1. **지도 라이브러리 도입**: 네이버지도, 카카오맵, 구글맵 등의 API를 로드하여 실제 동적 지도 캔버스 렌더링.
2. **좌표 투영 연동**: 모의 렌더링된 마커 배열 렌더러를 API에서 제공하는 정규 `CustomOverlay` 또는 `Marker` 객체로 치환.
3. **panTo 연동**: 리스트에서 카드를 클릭했을 때 브라우저 뷰포트 스크롤만 움직이는 것이 아니라, API 함수인 `map.panTo(Lat, Lng)`를 호출하여 실제 지도의 앵글도 중심 위치로 스무스하게 날아가도록 고도화 필요.
