# 동플허브_프론트구조_구현

## 1. 문서 목적
- 이 문서는 구현팀이 프론트 저장소를 어떤 구조로 시작해야 하는지 정하는 기준 문서다.
- 기존 문서들이 참조하는 `src/...` 경로를 실제 구현 가능한 도메인 구조로 재정리한다.

## 2. 프론트 구조 원칙
- 페이지 기준보다 도메인 기준으로 구조를 나눈다.
- UI 컴포넌트와 데이터 접근 로직을 분리한다.
- 지도, 상태 보드, 메인페이지, 이벤트 목록은 공통 타입과 서비스 계층을 공유한다.

## 3. 권장 폴더 구조

## 3.1 전역 내비게이션(하단 메뉴) 기준
- 하단 메뉴의 기본 축은 `홈(상황판) / 제보 / 지역 / 알림 / 나의 동플`로 유지한다.
- `앨범`은 하단 메뉴의 독립 탭으로 두지 않고, `나의 동플` 내부 기능으로 편입한다.
- `나의 동플` 내부 권장 구성:
  - 내 활동 요약
  - 내 인증/설정
  - 내 알림 설정
  - 내 앨범(여정의 앨범)

## 4. 페이지 책임

### 4.1 메인페이지
- 역할: 서비스 첫 진입점
- 주요 구성:
  - 핵심 CTA
  - 실시간 상태 요약
  - 주요 이벤트 요약
  - 지역 기반 탐색 유도
- 편성 원칙:
  - 사용자 참여형 상황(실시간 상태/제보) 콘텐츠를 우선 배치한다.
  - 공식 이벤트/지역 소식 콘텐츠는 주민 관심 유지를 위해 상시 노출한다.
  - 권장 초기 비중은 상황 60~75%, 이벤트 25~40%로 시작하고 제보량에 따라 탄력 조정한다.
- 관련 문서:
  - `동플허브_메인페이지_보완.md`
  - `동플허브_V2메인전환_계획.md`

### 4.2 지도 페이지
- 역할: 위치 기반 상태 탐색 + 실시간 공유 메인 허브
- 주요 구성:
  - 지도
  - 실시간 공유 스트림(상태 카드)
  - 바텀시트 리스트
  - 상태 제보/요청 액션
- 관련 문서:
  - `동플허브_지도화면_보완.md`
  - `동플허브_실시간상태보드_보완.md`

### 4.3 상태 보드
- 역할: 지도 화면 내부 패널 컴포넌트(독립 주메뉴 아님)
- 주요 구성:
  - 카드 목록
  - `맞아요` / `아니에요`
  - 상세 메시지 모달
- 관련 문서:
  - `동플허브_실시간상태보드_보완.md`

### 4.4 이벤트 페이지
- 역할: TourAPI 기반 공식 이벤트 노출
- 주요 구성:
  - 리스트/카드
  - 상세 정보
  - 공식 인증 뱃지

### 4.5 커뮤니티 페이지
- 역할: 게시글 목록/상세/작성
- 주요 구성:
  - 익명/비익명 작성 폼
  - 게시글 피드

## 5. 컴포넌트 설계 원칙
- `page.tsx`는 조립만 하고, 실제 동작은 `features/*`에서 담당한다.
- 카드, 모달, 바텀시트 같은 재사용 UI는 feature 내부 또는 `shared/ui`로 분리한다.
- 지도 라이브러리 연결은 지도 페이지 전용 어댑터로 감싼다.

## 6. 서비스 설계 원칙

### statusService.ts
- 상태 목록 조회
- 상태 등록
- 인증 수 증가
- 지도용 데이터 조회

### eventService.ts
- 이벤트 목록 조회
- 이벤트 상세 조회
- TourAPI 연동 결과 매핑

### postService.ts
- 게시글 목록/상세/등록
- 익명 표시 로직 변환

### authService.ts
- 사용자 세션 확인
- 익명/닉네임 노출 정책 보조

## 7. 구현 순서
1. `status` 도메인부터 만든다.
2. `map` 페이지와 `LiveStatusBoard`를 연결한다.
3. `events` 도메인을 추가한다.
4. `posts` 도메인을 추가한다.
5. 마지막에 메인페이지와 V2 전환을 정리한다.

## 8. 홈 피드 노출 정책(운영 가이드)
- 홈은 “동네 상황”과 “동네 주요 소식”을 함께 보여주는 혼합 피드로 운영한다.
- 기본 우선순위는 실시간 상태 카드이며, 공식 이벤트 카드는 보조 축으로 유지한다.
- 피드가 비는 시간대/지역에서는 이벤트 카드 비중을 높여 연속 탐색 경험을 보장한다.
- 실시간 공유의 상세 탐색 및 제보는 홈이 아닌 지도 화면으로 연결한다.

## 9. 기존 문서와의 매핑
- 기존 `src/components/dashboard/LiveStatusBoard.tsx`는 `features/status/components/LiveStatusBoard.tsx` 기준으로 재배치한다.
- 기존 `src/services/statusService.ts`는 `services/statusService.ts`로 유지 가능하다.
- 기존 `src/app/map/page.tsx`는 그대로 유지 가능하되, 내부 로직은 `features/status`, `features/events`에서 가져오게 한다.
- 기존 `src/app/v2/page.tsx` 구조가 있다면 전환 완료 후 `src/app/page.tsx`로 승격한다.
