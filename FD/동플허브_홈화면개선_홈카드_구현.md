# 동플 홈 카드 슬라이더 UX 설계서

---

## 1. 목적

홈 화면에서 유저가 여러 장소의 상태를 빠르게 비교하고
**“지금 어디를 갈지 / 피할지” 판단하도록 만든다.**

핵심 개념:

* 콘텐츠 소비 ❌
* 선택 유도 ⭕

---

## 2. 홈 전체 구조

```txt
홈

1. 상단 카피
   “지금 어디가 살아있을까?”

2. 🔥 지금 붐비는 곳
   [가로 슬라이더]

3. 🟢 지금 가기 좋은 곳
   [가로 슬라이더]

4. 🎉 오늘 행사 상황
   [가로 슬라이더]

5. 📍 내 주변 상태 요약

6. ⚡ 지금 올라오는 상황
   [실시간 상황판]
```

---

## 3. 슬라이더 카드 구조

### 3.1 기본 장소 카드

```txt
행궁동 화성공원

🔴 지금 붐빔
3분 전 업데이트

대기 김 / 사람 많음

[지도 보기]
```

---

### 3.2 행사 카드

```txt
🎉 벚꽃축제

🔴 지금 붐빔
5분 전 업데이트

입장 대기 김 / 주변 혼잡

[지도 보기]
```

---

## 4. 카드 디자인 규칙

### 4.1 카드 너비

```txt
width: 82~88%
```

* 옆 카드가 일부 보이도록 구성
* “더 넘길 수 있음”을 시각적으로 전달

---

### 4.2 정보 우선순위

```txt
1. 장소명
2. 상태 (가장 강조)
3. 업데이트 시간
4. 태그
5. 버튼
```

---

### 4.3 상태 색상

```txt
붐빔 → 빨강 (#FF4D4F)
보통 → 노랑 (#FAAD14)
한산 → 초록 (#52C41A)
```

* 상태 색상이 가장 먼저 인지되어야 한다

---

## 5. 섹션별 데이터 기준

---

### 5.1 🔥 지금 붐비는 곳

정렬 기준:

```txt
1. 붐빔 상태
2. 최신 업데이트
3. 리포트 수
```

노출 개수:

```txt
최대 3~5개
```

---

### 5.2 🟢 지금 가기 좋은 곳

정렬 기준:

```txt
1. 한산 상태
2. 최신 업데이트
3. 긍정 태그 수
```

긍정 태그 예시:

```txt
조용함
자리 있음
이동 편함
나들이 좋음
```

---

### 5.3 🎉 오늘 행사 상황

정렬 기준:

```txt
1. 현재 진행 중 행사
2. 실시간 리포트 존재 여부
3. 최신 업데이트
```

---

### 상태 없는 행사 처리

```txt
벚꽃축제

아직 현장 공유 없음

[첫 상태 남기기]
```

---

## 6. 카드 인터랙션

---

### 6.1 카드 클릭

```txt
홈 카드 클릭
→ 지도 이동
→ 해당 위치로 카메라 이동
→ 마커 강조
→ 바텀시트 자동 오픈
```

---

### 6.2 버튼 클릭

```txt
[지도 보기] 클릭
→ 카드 클릭과 동일 동작
```

---

### 6.3 공유 버튼 정책

홈에서는 공유를 최소화한다.

```txt
홈 = 판단
지도 = 행동
```

필요 시:

```txt
이 장소 상태 공유
```

으로 명확히 구분

---

## 7. 빈 상태 처리

---

### 붐비는 곳 없음

```txt
아직 붐빈 곳이 없어요
지금 주변 상태를 공유해보세요
```

---

### 가기 좋은 곳 없음

```txt
아직 한산한 곳이 없어요
주변을 확인해볼까요?
```

---

### 행사 데이터 없음

```txt
오늘 행사가 없어요
```
