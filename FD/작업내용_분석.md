# 동네 커뮤니티(Dongple) 작업 내용 분석

현재 프로젝트의 주요 파일들을 분석한 결과, 동네 기반의 커뮤니티 및 정보 제공 플랫폼(예: 수원 정자동, 역삼동 등)을 개발 중인 것으로 파악됩니다. 

## 1. 현재 구현 상태 (Mock Data 기반 UI 완성 단계)
- **`src/services/api.ts`**
  - 카카오 로컬 API(좌표-주소 변환, 주변 장소 검색) 및 공공데이터 기상청 API 연동을 위한 인터페이스와 함수 뼈대가 구현되어 있습니다.
  - 현재는 실제 API 호출 대신 `console.log` 출력과 함께 하드코딩된 모의(Mock) 데이터를 반환하도록 임시 처리되어 있습니다.
- **`src/components/dashboard/QuestionSummary.tsx`**
  - 'AI 동네 브리핑' 컴포넌트로, 동네 질문 트렌드, 답변 해결률, 실시간 교통 주의 등 통계 정보를 시각적으로 보여줍니다.
  - 이 역시 백엔드 연동 없이 정적인 모의 데이터(`stats` 배열)를 사용하여 UI 레이아웃과 디자인 디테일(Tailwind CSS)을 잡은 상태입니다.
- **`src/components/dashboard/LiveStatusBoard.tsx`**
  - 실시간 동네 상황(공원 여유, 관공서 대기, 교통 혼잡 등)을 보여주고 주민들이 참여해 인증하거나 요청할 수 있는 기능을 담은 컴포넌트입니다.
  - 클라이언트 사이드(`"use client"`) 컴포넌트로, `liveUpdates` 상태 값을 모의 데이터로 초기화하여 사용 중이며, `handleVerify` 함수를 통해 사용자의 인터랙션(인증 클릭 등)에 따른 화면 업데이트 UI 로직만 시뮬레이션하고 있습니다.

## 2. 향후 진행(작업할) 내용 가이드

위의 분석을 토대로, 앞으로 진행해야 할 주요 개발 작업은 다음과 같습니다.

### API 및 백엔드 연동 작업
1. **외부 API 실제 연동 (`api.ts`)**
   - 카카오 REST API 키를 발급받아 `.env.local` 에 설정하고, `getAddressFromCoords`, `getNearbyPlaces`에 실제 HTTP 요청 로직(예: `axios` 또는 `fetch`)을 구현해야 합니다.
   - 기상청 단기예보 API 등 공공데이터 포털을 통한 날씨 정보 수집 로직을 구현합니다.
2. **백엔드 서버 API 연결 (`QuestionSummary.tsx`, `LiveStatusBoard.tsx`)**
   - 현재 프론트엔드에 하드코딩된 `stats`, `liveUpdates` 데이터를 백엔드 서버(DB)에서 동적으로 불러오도록 비동기 데이터 패칭(Fetching) 로직으로 교체해야 합니다. 
   - React Query(또는 SWR) 등의 라이브러리를 도입하여 서버 상태 관리를 적용하는 것이 좋습니다.

### 동적 기능 및 상태 관리 로직 고도화
1. **실시간/사용자 인터랙션 고도화 (`LiveStatusBoard.tsx`)**
   - `handleVerify` 실행 시, 프론트엔드 UI 상태만 변경하는 것을 넘어 백엔드 서버에 실제 상태 업데이트(인증 누적, 요청 답변 완료) 요청(POST/PUT/PATCH)을 보내도록 합니다.
   - 웹소켓(WebSocket)이나 SSE(Server-Sent Events)를 검토하여 '실시간' 성격을 더 강화할 수 있습니다.
2. **에러 핸들링 및 로딩 상태 처리**
   - API 연동 전이므로 로딩 스피너 및 오류 발생 시 UI(Fallback UI)가 부족합니다. 스켈레톤 UI 또는 에러 바운더리 등을 추가 적용해야 합니다.

### 환경 변수 셋팅
- 열려있던 `.env.local.example` 파일을 바탕으로 개발에 필요한 로컬 환경변수 파일(`.env.local`)을 셋팅하고 카카오 및 공공 API 키값 등을 주입하는 작업이 필수적입니다.
